1. MATRIZ DE CONFUSIÓN + MÉTRICAS

Datos:
TP = 50
FP = 10
FN = 5
TN = 35

Matriz de confusión:

               Pred: Sí    Pred: No
Real: Sí          50           5
Real: No          10          35

Cálculos:

Accuracy = (TP + TN) / Total
= (50 + 35) / (50+10+5+35) = 85/100 = 0.85

Precision = TP / (TP + FP)
= 50 / (50+10) = 50/60 = 0.83

Recall = TP / (TP + FN)
= 50 / (50+5) = 50/55 = 0.91

F1-score = 2 * (Precision * Recall) / (Precision + Recall)
= 2 * (0.83 * 0.91) / (0.83 + 0.91) ≈ 0.87

Discusión:
En detección de enfermedad, el Recall es más importante porque se busca detectar la mayor cantidad de enfermos (minimizar falsos negativos).

2. MÉTRICAS CON DESBALANCE DE CLASES

Matriz:

               Pred: Sí    Pred: No
Real: Sí          8           2
Real: No         90          900

Accuracy = (8+900)/1000 = 908/1000 = 0.908

Recall = 8 / (8+2) = 0.8

Precision = 8 / (8+90) = 8/98 ≈ 0.082

Respuestas:
1. No, Accuracy no es buena métrica porque el dataset está desbalanceado.
2. Recall = 0.8, Precision ≈ 0.082
3. Se prioriza Recall (detectar enfermos)
4. No es buen modelo porque tiene muchos falsos positivos.

3. ERROR EN REGRESIÓN

Datos:
Real: 100, 200, 300
Pred:  90, 210, 1000

Errores:
|100-90| = 10
|200-210| = 10
|300-1000| = 700

MAE = (10+10+700)/3 = 720/3 = 240

MSE = (10^2 + 10^2 + 700^2)/3
= (100 + 100 + 490000)/3 = 490200/3 ≈ 163400

Reflexión:
El MSE penaliza más los errores grandes.

8. NAIVE BAYES

Datos:
P(A|Spam)=0.6
P(B|Spam)=0.7
P(Spam)=0.4

P(A|NoSpam)=0.2
P(B|NoSpam)=0.3
P(NoSpam)=0.6

Cálculos:

P(Spam|datos) ∝ 0.4 * 0.6 * 0.7 = 0.168
P(NoSpam|datos) ∝ 0.6 * 0.2 * 0.3 = 0.036

Normalización:
Total = 0.168 + 0.036 = 0.204

P(Spam|datos) = 0.168 / 0.204 ≈ 0.82
P(NoSpam|datos) = 0.036 / 0.204 ≈ 0.18

Clasificación:
Spam

Reflexión:
La independencia no siempre es real, pero funciona bien en práctica.
